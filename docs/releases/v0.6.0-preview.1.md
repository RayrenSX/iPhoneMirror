## Highlights

- USB wired capture and local-network AirPlay now share the same multi-device
  session, preview, audio, screenshot, detached-window and OBS paths.
- Three per-device wired modes are available: A Demo (`Valeria=true`,
  recommended), B AirPlay (experimental) and C Aisi-compatible fixed-size mode.
- Wireless AirPlay is isolated in a GPLv3 host process and does not require or
  inspect the USB capture driver.

## Driver management

- The main app no longer installs or mutates drivers.
- `iPhoneMirror.Driver.exe` is distributed as an independent executable and can
  install Apple USB support, install/repair/remove the signed libusb-win32
  1.2.6.0 filter, preserve snapshots and write operation logs.
- The main window exposes a Driver manager button.
- A wired Start action performs an exact-device preflight. Missing, invalid,
  unloaded or unverifiable driver state blocks the session and opens the manager.
- The driver manager uses signed offline Apple MSI payloads when available and
  Apple's official iTunes installer fallback when needed. It does not use
  private Aisi driver binaries.

## UI and capture

- Add stable multi-device tabs and simultaneous detached previews.
- Add per-window mute and mute-other-windows context actions.
- Add device-aware rounded preview outlines, rotation, fullscreen and local
  render/FPS limits.
- Add bounded wireless IPC, H.264/NV12 conversion, AirPlay lifecycle tests and
  host-stop cleanup.
- Add unified Chinese/English documentation and release packaging.

## Dependencies and release contents

- `iPhoneMirror.exe`: self-contained .NET/WPF main app.
- `iPhoneMirror.Driver.exe`: independent driver manager.
- `iPhoneMirror.Core.dll`: C++20 USB/QuickTime/AirPlay session core.
- `Wireless/`: isolated AirPlayServer/FFmpeg runtime and license files.
- `THIRD_PARTY_NOTICES.md`: source, license and hash obligations for all vendored
  components.

The release does not redistribute Apple Devices, Apple Mobile Device Support or
iTunes. Those are installed from Apple/Microsoft sources by the driver manager
when the target machine needs them.
