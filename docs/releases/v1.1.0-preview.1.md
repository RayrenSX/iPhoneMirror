# iPhoneMirror v1.1.0-preview.1

预览版，面向 Windows 10/11 x64。发布日期：2026-07-17。

这是 1.1 系列的第一个测试版，重点加入视频应用投屏，并整理了无线接收、设备会话、
原生预览和发布打包的生命周期边界。

## 视频应用投屏

- 新增“视频应用投屏”接收模式，可接收视频 App 通过 HTTP(S) 或 HLS 地址发送的影片。
- 屏幕镜像与视频投放向用户显示同一个 AirPlay 接收端名称；程序按请求类型分流，避免用户
  在手机端面对两个难以区分的目标。
- 视频播放请求进入主窗口内的专用播放界面，不会被注册成屏幕镜像设备，也不会覆盖已有的
  USB 或 AirPlay 镜像会话。
- 播放时长、当前位置和播放状态会通过双向命名管道反馈给无线宿主，再返回发送设备。
- 新增轻量 DLNA/UPnP MediaRenderer，提供 SSDP 发现、设备描述、AVTransport、
  ConnectionManager 和 RenderingControl 基础动作，兼容使用投屏按钮而非控制中心镜像的 App。

## 会话、窗口与无线接收重构

- 将设备会话、屏幕镜像接收器和视频投放接收器的启动、停止与销毁职责拆分为独立服务，
  减少主 ViewModel 中重复的生命周期分支。
- 删除旧的 XAML 独立预览实现，统一使用原生预览窗口；多设备、旋转、圆角、比例锁定和
  全屏继续通过 SessionHandle 与 HWND 绑定。
- 屏幕镜像和视频投放共用一个 combined-mode 无线宿主和接收端身份，通过有界 IPC
  消息类型分流；镜像设备会话与视频播放界面仍保持独立状态。
- 强化命名管道客户端进程验证、IPC 长度限制、父进程退出检测与宿主清理。
- 修复 DLNA HTTP/SOAP 请求被拆分到多个 TCP 数据包时接收端提前断开连接的问题。
- 新增有界 UTF-8 原生日志增量读取，避免实时日志面板重复扫描完整日志文件。

## 构建、打包与依赖

- 主程序和驱动管理器改为压缩的自包含单文件 .NET 发布，目标仍为 `win-x64`。
- C++ Core、WirelessHost、DNS-SD shim、AirPlayServer 和 FFmpeg 继续作为外部运行时文件
  随包分发，便于确定性加载并保留完整许可证材料。
- 更新固定版本 AirPlayServer 的屏幕镜像补丁、构建校验、来源说明和 SHA-256 清单。
- 发布包继续排除日志、转储、抓包和 PDB，并提供 SPDX SBOM 与 SHA-256 校验文件。

## 已知限制

- 这是预览版；视频应用投屏的 App 兼容范围仍需扩大真机测试。
- DRM、登录态、Cookie 或私有播放协议保护的内容可能不会向第三方接收端提供可播放 URL。
- 当前跨进程播放控制以 Play、Stop、Seek 和状态查询为主；部分发送端的 Pause 行为可能
  只更新接收端状态，尚未完整映射到 WPF 播放器。
- 实际容器、编码和 HLS 播放能力取决于 Windows MediaElement 与系统媒体组件。
- Apple USB、QuickTime 与兼容 AirPlay 均不是 Apple 的稳定第三方 API，未来 iOS 更新可能
  需要继续适配。
- 程序尚未进行商业 Authenticode 签名，干净系统和更广泛网络环境仍需继续验证。
