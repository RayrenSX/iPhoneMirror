# iPhoneMirror v1.0.0

发布日期：2026-07-14

这是 iPhoneMirror 的首个正式版本。USB 有线采集与本地网络 AirPlay
共用同一套多设备会话、预览、音频、截图、独立窗口和 OBS 通道。

  `THIRD_PARTY_NOTICES.md`。

## 亮点

- 支持 USB 有线投屏和本地网络 AirPlay 无线投屏。
- 多台 iPhone 可以同时连接，并拥有独立会话、预览窗口和音频状态。
- AirPlay 服务默认启动，仅在设备实际连接后显示无线设备条目。
- 有线和无线来源统一接入截图、旋转、全屏、独立窗口及 OBS 窗口采集流程。
- 无线 AirPlay 在隔离的接收进程中运行，不要求也不会检查 USB 采集驱动。

## 有线投屏模式

每台有线设备可以独立选择以下模式，开始投屏后会隐藏模式选择：

- **A 演示模式（推荐）**：发送 `Valeria=true`，优先获得清晰、完整的镜像画面；
  缺点是设备状态栏的日期和时间可能被锁定。
- **B AirPlay（实验）**：发送 `Valeria=false` 并使用自适应横竖屏配置，允许视频应用
  进入 AirPlay 内容播放；部分内容的分辨率可能无法完美自适应，出现裁切或显示不全。
  高级模式允许自定义参数，但错误设置可能导致黑屏或连接异常。
- **C 爱思模式**：使用固定兼容尺寸以提高部分应用的兼容性；缺点是会限制输出分辨率，
  画面清晰度通常低于演示模式。

## 驱动管理

主程序不直接安装或修改驱动。发布包提供独立的
`iPhoneMirror.Driver.exe` 驱动管理器，支持：

- 一键检测并安装缺失的 Apple USB 支持和设备级 `libusb-win32 1.2.6.0` 过滤驱动。
- 检测错误的父驱动配置，执行修复、卸载、重新安装和结果验证。
- 自动识别设备型号、设备名称、系统版本和序列信息。
- 保存驱动状态快照、操作日志和失败诊断信息。
- 仅在确实需要信任、拔线、重新插线等人工操作时弹出提示。

有线模式点击开始投屏时会执行精确设备预检查。驱动缺失、无效、未加载或无法验证时，
会阻止该有线会话并打开驱动管理器；无线 AirPlay 不参与此项检查。

驱动管理器优先使用可用的签名 Apple MSI 安装包，并在需要时回退到 Apple 官方
iTunes 安装程序。项目不分发私有爱思驱动二进制文件。

遇到无法自动解决的驱动问题时，可以加入 QQ 群 **1050045279**；也可以先使用
爱思投屏完成一次正常有线投屏，让系统建立所需驱动状态后再运行 iPhoneMirror。

# iPhoneMirror v1.0.0

Release date: 2026-07-14

This is the first stable iPhoneMirror release. USB capture and local-network
AirPlay share the same multi-device session, preview, audio, screenshot,
detached-window and OBS pipelines.

## Highlights

- USB wired mirroring and local-network AirPlay wireless mirroring.
- Multiple iPhones can connect simultaneously with independent sessions,
  preview windows and audio state.
- The AirPlay service starts automatically; wireless devices appear only after
  they connect.
- Wired and wireless sources share screenshots, rotation, fullscreen,
  detached-window and OBS Window Capture workflows.
- Wireless AirPlay runs in an isolated receiver process and does not require or
  inspect USB capture drivers.

## Wired modes

Each wired device has its own mode selection, which is hidden while its session
is running:

- **A Demo mode (recommended)** sends `Valeria=true` and prioritizes a clear,
  complete mirrored image. The device status-bar date and time may be locked.
- **B AirPlay (experimental)** sends `Valeria=false` with adaptive orientation
  settings and allows video apps to enter AirPlay content playback. Some content
  may be cropped or fail to fit perfectly. Advanced custom settings are
  available, but invalid values can cause a black screen or connection failure.
- **C Aisi mode** uses a fixed compatibility size. It can improve compatibility
  for some apps, but limits output resolution and is usually less sharp than
  Demo mode.

## Driver management

The main app does not directly install or modify drivers. The package includes
the standalone `iPhoneMirror.Driver.exe` manager, which can:

- Detect and install missing Apple USB support and the per-device signed
  `libusb-win32 1.2.6.0` filter driver in one operation.
- Detect an invalid parent-driver configuration, then repair, uninstall,
  reinstall and verify it.
- Identify devices by model, name, OS version and serial information.
- Preserve driver snapshots, operation logs and failure diagnostics.
- Prompt only when trust, cable removal or reconnection is actually required.

Starting a wired session performs an exact-device preflight. A missing, invalid,
unloaded or unverifiable driver blocks that wired session and opens the manager.
Wireless AirPlay is never covered by this check.

The manager prefers an available signed Apple MSI payload and falls back to
Apple's official iTunes installer when required. Private Aisi driver binaries
are not distributed.

For unresolved driver failures, join QQ group **1050045279**. Another recovery
path is to complete one successful wired session with Aisi Screen Mirroring so
Windows establishes the required driver state, then run iPhoneMirror again.

## UI and capture
